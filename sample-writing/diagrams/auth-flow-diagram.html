Authentication & Authorization Flow

1
User Attempts Access
A user tries to access a system, application, or resource
Hotel Example:
Sarah arrives at the hotel and wants to access her room
↓
2
Authentication: "Who Are You?"
System verifies the user's identity through credentials
Hotel Example:
Sarah shows her ID and confirmation number at the front desk
↓
✗
Authentication Fails
Access denied - user cannot proceed
Hotel Result:
"No reservation found" - Sarah is asked to verify her booking
✓
Authentication Succeeds
Identity verified - proceed to authorization
Hotel Result:
Front desk confirms Sarah's identity and gives her a key card
↓
3
Authorization: "What Can You Do?"
System checks user's permissions and roles for the requested resource
Hotel Example:
Sarah tries to use her key card to access the penthouse suite
↓
✗
Authorization Fails
Access denied - user lacks permission
Hotel Result:
Red light - Sarah's key card doesn't work on the penthouse
✓
Authorization Succeeds
Access granted - user can proceed
Hotel Result:
Green light - Sarah's key card opens her assigned room
!
Key Point: Continuous Process
Authorization happens every time a user tries to access something new, even within the same session
Hotel Example:
Sarah can access her room and the gym, but not the executive lounge or other guests' rooms
